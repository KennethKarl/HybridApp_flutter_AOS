# 크로스 플랫폼 — 릴리즈 조율

> **목적**: Android와 iOS를 동시에 릴리즈해야 하는 경우의 조율 절차를 정의한다.
> **적용 시점**: 동시 릴리즈가 필요한 경우에만 사용. 각 플랫폼 독립 릴리즈가 가능하면 각 프로젝트의 세션 6 가이드만 따른다.

---

## 동시 릴리즈 절차

```
1. 릴리즈 일정 합의 (PM 주도)
2. 양 플랫폼 Feature Parity 확인 (feature-parity.md)
3. 각 플랫폼 독립 배포 체크리스트 수행 (각 프로젝트 세션 6)
4. 스테이징 배포 (양 플랫폼)
5. QA 검증 (양 플랫폼 동시)
6. 프로덕션 배포:
   - Play Store 릴리즈 (Android 세션 6)
   - App Store Connect 릴리즈 (iOS 세션 6)
7. 배포 후 모니터링 (양 플랫폼)
8. 릴리즈 완료 기록
```

---

## 버전 관리

| 항목 | 규칙 |
|------|------|
| 버전 번호 | Android와 iOS 동일 버전 사용 (예: v1.2.0) |
| 버전 코드 | 각 플랫폼 독립 관리 (Android versionCode, iOS build number) |
| CHANGELOG | 각 플랫폼 독립 작성 |
